== Introduction [[Introduction]]

This use case shows how to use the Axelor REST API to retrieve data from an Axelor instance. For details on the REST API, refer to the official Axelor documentation.

== Goal: Fetch a Specific Product [[Goal]]

In this example we retrieve a product by sending a request to an Axelor instance. It is a xref:general-documentation/web-services/ws-connectors.adoc[connector] with a single xref:general-documentation/web-services/ws-requests.adoc[request] of type `GET` that fetches a product by its ID.

The web service is accessible with the following URL pattern:

----
GET /ws/rest/:model/:id
----

.Request
----
GET /ws/rest/com.axelor.apps.base.db.Product/1 HTTP/1.1
Accept: application/json
----

== Authentication [[Authentication]]

To connect to an Axelor instance, create an xref:general-documentation/web-services/ws-authenticators.adoc[authenticator]. This is a custom authentication that requires a `username` and a `password`.

== Get Product Request [[Get-Product]]

To get a product by ID, send a request to:

----
GET : baseUrl/com.axelor.apps.db.Product/${productID}
----

image::web-services/getProduct.png[Get product request,align="left"]

TIP: You can provide the product ID through the connector context by passing it from the BPM connector script. Every dynamic value is written as `${...}` and is processed as a Groovy template.
